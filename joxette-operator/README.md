# joxette-operator

Kubernetes operator for Joxette, built with the [Java Operator SDK](https://javaoperatorsdk.io/)
(JOSDK 5.3.4 + Fabric8 7.7.0) on the Spring Boot 4 starter. Full design:
[`docs/operator-design.md`](../docs/operator-design.md).

## Status — Phase 1 (in progress)

Implemented:
- **`JoxetteCluster` CRD** (`joxette.dev/v1alpha1`) — spec mirrors the Helm chart
  values; CRD YAML generated by the Fabric8 crd-generator into
  `target/classes/META-INF/fabric8/joxetteclusters.joxette.dev-v1.yml`.
- **`JoxetteClusterReconciler`** — runs the catalog guardrail, then server-side
  applies the desired workloads (owned by the CR for cascading delete), then
  writes `.status` (`Ready` / `Rejected`).
- **`CatalogGuardrail`** — rejects unsafe specs (embedded + replicas>1 / split
  scale-out / pekko-management, or a shared backend without a URI), mirroring the
  Helm chart's `joxette.validate` and design doc §5.
- **`JoxetteClusterResources`** — builds the Fabric8 objects: embedded → one
  all-in-one StatefulSet + PVC; shared backend → per-tier Deployments (recorder /
  replay / compaction pinned to 1); plus the main Service and, in
  pekko-management mode, a headless Service.

Not yet (later phases): the admission webhook, `/health` status polling, the pod
RBAC + ConfigMap as dependent resources, and the `RecordedTopic` / `EntityType`
reconcilers (Phase 2).

## Build & run

```bash
# unit tests (guardrail, resource builder, context load)
mvn -pl joxette-operator test

# generate the CRD YAML
mvn -pl joxette-operator process-classes
#  -> target/classes/META-INF/fabric8/joxetteclusters.joxette.dev-v1.yml

# operator image (buildpacks)
mvn -pl joxette-operator spring-boot:build-image
```
